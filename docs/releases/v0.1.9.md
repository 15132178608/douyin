# v0.1.9

This release adds startup preflight checks for the Windows installer launcher.

## What changed

- The launcher now runs 启动前健康检查 before downloading runtime dependencies.
- It verifies that the install directory, `data\logs`, and `D:\codexDownload\douyinclaude-runtime` are writable.
- If uv is not already installed, it checks that `https://astral.sh/uv/install.ps1` is reachable before trying to download it.
- Preflight failures now stop early with Chinese fix hints for permissions, `D:\codexDownload` write access, network, proxy, or firewall problems.

## Install notes

- The installer is still unsigned. Windows SmartScreen may warn on first download. 当前安装包仍然未签名。
- First launch can download Python dependencies, Playwright browser assets, and local model files. 首次启动可能需要较长时间。
- Runtime downloads and caches remain under `D:\codexDownload\douyinclaude-runtime`.
- The launcher still sets `UV_LINK_MODE=copy` to avoid cross-drive hardlink warnings.
- Use `uv run recall stop` when you want to stop the local background web service after development, testing, or before reinstalling.
- Maintenance remains available at `/maintenance`.
