# v0.1.6

This release tightens the Windows installer recovery path.

## What changed

- The installer launch script now writes a dedicated startup log at `data\logs\start-douyin-recall.log`.
- If startup fails, the launch window now prints the most useful recovery commands:
  - `uv run recall status`
  - `uv run recall stop`
  - `uv run recall diagnose`
- The same failure output points to `/maintenance`, the service logs, the startup log, and the runtime cache at `D:\codexDownload\douyinclaude-runtime`.
- Added `docs/windows-troubleshooting.md` with Windows installer recovery guidance for SmartScreen warnings, first-run downloads, logs, stale background services, and diagnostics.

## Install notes

- The installer is still unsigned. Windows SmartScreen may warn on first download. 当前安装包仍然未签名。
- First launch can download Python dependencies, Playwright browser assets, and local model files. 首次启动可能需要较长时间。
- Runtime downloads and caches remain under `D:\codexDownload\douyinclaude-runtime`.
- Use `uv run recall stop` when you want to stop the local background web service after development or testing.
