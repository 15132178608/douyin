# v0.1.20

This release stabilizes the Windows installer path for personal local use.

## What changed

- First-run setup now opens directly to Douyin QR binding instead of showing an environment/sync/index wizard; after scanning, sync and indexing continue automatically in the background.
- The favorite and like pages now keep existing local items visible while background sync runs.
- Background sync banners now use consistent copy for 收藏 and 喜欢, with a small spinner, live status dot, and animated progress fill so the page no longer feels frozen.
- The progress banner no longer refreshes the full result area every few seconds, avoiding card/list flicker during long sync jobs.
- Installer Finish and the main app shortcuts run the launcher hidden in the background; users should only see the final local Web page after startup succeeds.
- The launcher waits until the local Web endpoint is actually reachable before opening `127.0.0.1`, avoiding early `ERR_CONNECTION_REFUSED` browser tabs.
- The installer always shows the installation directory page, including reinstall or upgrade flows.
- Windows startup writes a local runtime preparation status file at `data\runtime\startup-status.html` for diagnostics, but no longer opens that preparation page during normal startup.
- Windows startup and `Douyin Recall Prepare Runtime` print coarse step progress so long first-run downloads are easier to follow.
- `Douyin Recall Prepare Runtime` ends with a runtime preparation summary listing completed steps, install path, runtime cache, browser cache, logs, and next start/stop entries.
- Windows launcher and control shortcuts run `python -m src.cli` through uv, so installs under non-ASCII paths such as `D:\测试\DouyinRecall` do not fail with `ModuleNotFoundError: No module named 'src'`.
- Main shortcuts and uninstall metadata use a Douyin Recall icon instead of the default PowerShell icon.
- Main app shortcuts start through a tiny `wscript.exe` launcher, avoiding the brief PowerShell console flash when opening Douyin Recall from the desktop.
- Installed-build QA now saves and restores the Inno Setup registration after isolated test installs, so QA runs cannot change the user's real install location.

## Install notes

- The installer is still unsigned. Windows SmartScreen may warn on first download. 当前安装包仍然未签名。
- First launch can download Python dependencies, Playwright browser assets, and local model files. 首次启动可能需要较长时间。
- Runtime downloads and caches remain under `D:\codexDownload\douyinclaude-runtime`, with `UV_LINK_MODE=copy` for cross-drive installs.
- The launcher still runs 启动前健康检查 before preparing dependencies.
- Normal startup opens only the final local Web UI after the service is reachable; the local status file and logs are retained only for troubleshooting.
- The Start Menu 控制入口 remains `Douyin Recall Control`, and it still prints the 状态摘要 before the menu.
- Existing 健康检查 behavior remains available through `Douyin Recall Health Check`.
- Use `Douyin Recall Prepare Runtime` when first launch fails during uv setup, `uv sync`, Playwright Chromium install, or database initialization.
- `Douyin Recall Prepare Runtime` does not start the local web service, so it can be used as a safe retry step before opening the app.
- Existing local operations remain available: `Douyin Recall Stop Service`, `Douyin Recall Health Check`, and `Douyin Recall Repair State`.
- Backup and recovery entries remain available: `Douyin Recall Backup Now`, `Douyin Recall Backups`, `Douyin Recall Restore Center`, and `Douyin Recall Verify Backup`.
- Use `Douyin Recall Verify Backup` or `recall verify-backup` for a read-only recovery drill against the latest `data\exports` backup.
- Installer upgrade safety backups still use `pre-install-recall-*.db`.
- Use `uv run python -m src.cli stop` or `Douyin Recall Stop Service` when you want to stop the local background web service.
- Use `Douyin Recall Account Recovery` when `/maintenance` says the Douyin login state may be expired, or open `/auth` directly.
- Maintenance remains available at `/maintenance`.
