# v0.1.24

This patch makes installed-runtime diagnostics match the cache and service behavior shipped in v0.1.23. It does not change user content, search schema, or account data.

## Accurate model-cache diagnostics

- Doctor checks explicit `SENTENCE_TRANSFORMERS_HOME`, `HF_HOME`, `HF_HUB_CACHE`, `HUGGINGFACE_HUB_CACHE`, and `TRANSFORMERS_CACHE` locations first.
- On Windows it then checks `D:\codexDownload\douyinclaude-runtime\huggingface`, followed by the legacy project `data\models` directory for compatibility.
- A cache is reported as ready only when Doctor can read a non-empty model configuration, tokenizer, or weight file.
- Empty nested directories, metadata-only snapshots, lock trees, zero-byte markers, and `.incomplete`, `.part`, or `.tmp` downloads no longer produce false-positive cache reports.
- Snapshot scanning is bounded and supports direct Hugging Face, `hub`, and Sentence Transformers layouts without binding diagnostics to a single model name.

## Safe service-audit guidance

- `record_without_listener` and `record_port_mismatch` now use the Stop Service path because the recorded PID is still alive and Repair State intentionally refuses live-process cleanup.
- `stale_record` and `stale_record_with_listener` continue to use Repair State because their recorded PID is gone.
- Python CLI status, `Douyin Recall Control`, `Douyin Recall Health Check`, and the Windows troubleshooting guide now give the same recommendation.
- Mismatched port-owner guidance continues to warn against terminating an unknown process.

## Install notes

- The installer is still unsigned. Windows SmartScreen may warn on first download. 当前安装包仍然未签名。
- First launch can download Python dependencies, Playwright browser assets, and local model files. 首次启动可能需要较长时间。
- Runtime downloads and caches remain under `D:\codexDownload\douyinclaude-runtime`, with `UV_LINK_MODE=copy` for cross-drive installs.
- The launcher still runs 启动前健康检查 before preparing dependencies.
- Normal startup no longer opens that preparation page during normal startup; status files and logs remain available for troubleshooting.
- Installer Finish and the main shortcuts run the launcher hidden in the background.
- The launcher waits until the local Web endpoint is actually reachable before opening the browser.
- The installer always shows the installation directory page during fresh install, reinstall, and upgrade.
- The Start Menu 控制入口 remains `Douyin Recall Control`, and it prints a 状态摘要 before the menu.
- Existing 健康检查 and repair behavior remain available through `Douyin Recall Health Check` and `Douyin Recall Repair State`.
- Use `Douyin Recall Prepare Runtime` when dependency, browser, or database preparation fails; it does not start the local web service.
- Local operations remain available through `Douyin Recall Stop Service` and `uv run python -m src.cli stop`.
- Backup and recovery entries remain available: `Douyin Recall Backup Now`, `Douyin Recall Backups`, `Douyin Recall Restore Center`, `Douyin Recall Verify Backup`, and `Douyin Recall Rollback Check`.
- Use `Douyin Recall Verify Backup` or `recall verify-backup` for a read-only check of the latest backup in `data\exports`.
- Use `rollback-from-manifest` for a read-only delivery rollback check before explicitly applying any restore.
- Before `rollback-from-manifest --apply` targets the active database, stop the local Web service with `recall stop`; the CLI refuses a live-service restore, an active configured listener, or an acquired database runtime lock.
- Installer upgrade safety backups use the `pre-install-recall-*.db` filename pattern.
- Use `Douyin Recall Account Recovery` or `/auth` to rebind an account; maintenance remains available at `/maintenance`.

After all release gates pass, download `DouyinRecallSetup.exe` from the GitHub Release and install it over v0.1.23. The installer creates a pre-install SQLite backup before replacing application files.
