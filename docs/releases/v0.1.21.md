# v0.1.21

This release focuses on safe upgrades, background search recovery, Web security, and a fully verifiable Windows delivery path.

## Upgrade safety and search recovery

- Legacy vec/FTS tables are upgraded to user-partitioned schemas without leaving existing users with permanently empty search results.
- Schema migration writes durable per-user, per-content reindex markers in the same database transaction as the index rebuild.
- Web worker startup materializes those markers as idempotent `force=true` index jobs.
- A marker is completed only after active content, vector rows, and FTS rows match; failed work remains visible and retryable.
- Existing 收藏 and 喜欢 data remain local and are reindexed without requiring QR login or manual standard maintenance.

## Account and Web security

- Session lookup and default-user resolution run outside the async event loop.
- Invite claims now create the user, consume the invite, and create the session in one atomic SQLite transaction.
- Failed invite logins are rate-limited persistently by direct client IP and invite fingerprint.
- Public authenticated binds require HTTPS-secured session cookies.
- The avatar cache only accepts configured HTTPS CDN hosts, validates redirects and resolved addresses, enforces image type and size limits, and writes files atomically outside the event loop.

## Web architecture and onboarding

- `auth`, `setup`, `jobs`, `maintenance`, and `media` now own their route implementations.
- `src/web/app.py` is application assembly only; the wildcard import and dynamic compatibility shim are gone.
- Multi-account QR binding, account switching, background sync/index progress, and failure-safe public messages are covered by installed-build smoke tests.

## Operations and release verification

- Release Gate now covers a pre-release rollback point, full tests, doctor output, installed smoke, database safety, backup/restore, Web and SQL performance, acceptance evidence, and manifest rollback dry-run.
- `Douyin Recall Rollback Check` validates the latest delivery manifest without applying a restore.
- Delivery evidence and preflight summary reports make the final release decision auditable.
- Pytest is declared in the `dev` dependency group; installed Windows runtime preparation excludes development dependencies.

## Verification completed for the release candidate

- Full pytest suite passes.
- Fresh install and in-place upgrade are required before publishing the final GitHub release.
- The installer SHA256 is recorded by the installer-enabled Release Gate.

## Install notes

- The installer is still unsigned. Windows SmartScreen may warn on first download. 当前安装包仍然未签名。
- First launch can download Python dependencies, Playwright browser assets, and local model files. 首次启动可能需要较长时间。
- Runtime downloads and caches remain under `D:\codexDownload\douyinclaude-runtime`, with `UV_LINK_MODE=copy` for cross-drive installs.
- The launcher still runs 启动前健康检查 before preparing dependencies.
- Normal startup no longer opens that preparation page during normal startup; status files and logs remain available for troubleshooting.
- Installer Finish and the main shortcuts run the launcher hidden in the background.
- The launcher waits until the local Web endpoint is actually reachable before opening the browser.
- The installer always shows the installation directory page during fresh install, reinstall, and upgrade.
- The Start Menu 控制入口 remains `Douyin Recall Control`, and it prints a 状态摘要 before the menu.
- Existing 健康检查 and repair behavior remain available through `Douyin Recall Health Check` and `Douyin Recall Repair State`.
- Use `Douyin Recall Prepare Runtime` when dependency, browser, or database preparation fails; it does not start the local web service.
- Local operations remain available through `Douyin Recall Stop Service` and `uv run python -m src.cli stop`.
- Backup and recovery entries remain available: `Douyin Recall Backup Now`, `Douyin Recall Backups`, `Douyin Recall Restore Center`, `Douyin Recall Verify Backup`, and `Douyin Recall Rollback Check`.
- Use `Douyin Recall Verify Backup` or `recall verify-backup` for a read-only check of the latest backup in `data\exports`.
- Use `rollback-from-manifest` for a read-only delivery rollback check before explicitly applying any restore.
- Installer upgrade safety backups use the `pre-install-recall-*.db` filename pattern.
- Use `Douyin Recall Account Recovery` or `/auth` to rebind an account; maintenance remains available at `/maintenance`.

Download `DouyinRecallSetup.exe` from the GitHub Release and install it over the previous version to upgrade. The installer creates a pre-install SQLite backup before replacing application files.
