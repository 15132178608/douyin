# Douyin Recall v0.1.4

这一版把项目从“能开发运行”推进到“更适合长期本地安装使用”的状态，重点是维护、恢复、诊断和安装包首启体验。

## 主要变化

- 新增 `/maintenance` 维护中心，集中查看本地 Web 服务、同步、索引、失败任务和 SQLite 备份状态。
- 新增 SQLite 备份校验与恢复流程：恢复前做完整性检查，恢复时先生成恢复前安全备份。
- 新增 `recall status` 和 `recall stop`，并强化 Windows 停止逻辑，避免本地 Web 服务残留后台占用端口。
- 新增 `recall diagnose`，导出脱敏诊断包；排除 `.env`、数据库、浏览器 profile、登录态和 `data/users`。
- 安装包启动脚本会把运行时下载和缓存放到 `D:\codexDownload\douyinclaude-runtime`。
- 安装包排除规则更干净，不再创建空的 `.venv`、`data`、`.claude` 或缓存目录壳。

## 安装包说明

- 下载附件 `DouyinRecallSetup.exe` 后双击安装即可。
- 当前安装包未签名，Windows SmartScreen 可能提示风险；请只使用本 GitHub Release 页面下载的安装包。
- 首次启动会下载 Python 依赖和 Playwright 浏览器，耗时取决于网络。
- 首次生成搜索索引时会下载本地模型，耗时也会比较长。
- 本地 Web 服务默认运行在 `http://127.0.0.1:8000`。

## 常用维护命令

```powershell
uv run recall status
uv run recall stop
uv run recall diagnose
```

日常维护入口：

```text
http://127.0.0.1:8000/maintenance
```

## 数据安全

- 安装包不会包含你的 `.env`、SQLite 数据库、浏览器登录态或用户数据。
- 诊断包只包含脱敏后的环境、服务、任务和日志摘要。
- 卸载程序不会自动帮你批量删除个人数据；需要清理时请先确认备份。
