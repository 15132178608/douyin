# v0.1.12

This release adds local health check and safe stale-state repair entries for Windows installs.

## What changed

- Added `Douyin Recall Health Check` to run a 健康检查 without starting the local web service.
- Health check covers:
  - install directory write access
  - logs directory write access
  - `D:\codexDownload\douyinclaude-runtime` write access
  - uv availability
  - service record state from `data\runtime\server.json`
  - current port listener state
- Added `Douyin Recall Repair State` to clear stale service state when the recorded process is no longer running.
- Repair only removes explicit stale files: `data\runtime\server.json` and `data\runtime\server.pid`.
- `Douyin Recall Control` still prints the 状态摘要 before the menu.

## Install notes

- The installer is still unsigned. Windows SmartScreen may warn on first download. 当前安装包仍然未签名。
- First launch can download Python dependencies, Playwright browser assets, and local model files. 首次启动可能需要较长时间。
- Runtime downloads and caches remain under `D:\codexDownload\douyinclaude-runtime`, with `UV_LINK_MODE=copy` for cross-drive installs.
- The launcher still runs 启动前健康检查 before preparing dependencies.
- The Start Menu 控制入口 remains `Douyin Recall Control`.
- Use `Douyin Recall Stop Service` or `uv run recall stop` when you want to stop the local background web service after development, testing, or before reinstalling.
- Use `Douyin Recall Health Check` before troubleshooting startup problems.
- Use `Douyin Recall Repair State` only when health check reports a stale service record.
- Maintenance remains available at `/maintenance`.
