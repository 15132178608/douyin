## Completed in this pass

- Added a cross-process runtime lease for the live SQLite database. Web startup,
  live manifest rollback, server-state inspection, and stale-state cleanup now
  serialize on the same operating-system file lock.
- Kept the lease held through schema initialization, worker startup, worker
  shutdown, and restore cutover. Repeated asyncio cancellation cannot release the
  lease while a synchronous initialization or shutdown thread is still running.
- Moved lifecycle-state writes into the successful application lifespan and made
  cleanup ownership-aware, so a losing concurrent process cannot overwrite or
  erase the winning process state.
- Made `should_start_server` hold the runtime lease across state inspection,
  listener checks, and stale-state cleanup, closing the probe/release/clear race.
- Propagated both the effective host and effective port from the supported CLI
  startup path before Uvicorn starts.
- Added final-only worker shutdown semantics: final shutdown waits for the worker
  to exit and suppresses concurrent restarts, while maintenance-mode timeouts keep
  their existing recovery behavior.
- Serialized first-run authorization prewarm with final worker shutdown, so an
  older in-flight start cannot continue prewarming after shutdown has returned.
- Made every production service-state cleanup ownership-safe. Stop now acquires
  the runtime lease, re-reads the complete state snapshot, and preserves a
  starting or replacement instance instead of deleting its files.
- Added the lock-aware `repair-state` CLI command and routed the Windows Repair
  State start-menu action through it. Malformed stale JSON remains repairable,
  while a live fallback PID, held lock, or changed state is preserved.
- Blocked live `rollback-from-manifest --apply` when either the recorded service or
  the configured listener is active. Dry runs and explicitly targeted alternate
  databases remain available.
- Expanded sidecar quarantine/rollback double-failure diagnostics to preserve the
  exact forensic path and unique leaf errors from nested `ExceptionGroup` values.
- Added regression tests for all of the above concurrency, cancellation, restore,
  state-ownership, listener, and diagnostic paths.
- Updated the changelog and v0.1.23 release notes.

## Known non-blocking follow-ups

- The Douyin authorization daemon thread is not joined during graceful shutdown,
  leaving a very narrow unlock-to-process-exit window. Normal managed stop uses
  process termination and is not affected.
- `server.pid` and `server.json` are still two separate, non-atomic file writes.
  A partial write can leave stale state, which the next lock-protected startup
  repairs.
- Direct custom-port `uvicorn` invocation bypasses the supported CLI host/port
  synchronization. The official launcher and CLI path are correct.
- The release-check installed-smoke helper still performs direct state-file
  cleanup inside its isolated QA-only app root; no production entry point uses
  that path.
- The third-party jieba dependency still emits a `pkg_resources` deprecation
  warning.
